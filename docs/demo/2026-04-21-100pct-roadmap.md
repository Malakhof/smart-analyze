## Универсальный workflow для каждого нового клиента

1. **Получить OAuth credentials** (client_id, secret, auth_code, subdomain) от клиента
2. **Exchange auth_code → tokens** (СРАЗУ — auth_code 20 мин)
3. **create-tenant скрипт** (по примеру `create-shumoff.ts`): tenant + user + crmconfig
4. **Запустить smoke-sync** (или sync-engine)
5. **АУДИТ-СНАПШОТ** до показа клиенту: deals counts, funnels, managers, calls, messages
6. **Анкета (4 кита)** клиенту — с конкретными цифрами из аудита
7. После ответов — backfill scripts + калибровка
8. **Cron** каждые 2ч + post-sync pipeline (backfill, transcribe, score, insights)

## ВАЖНО: AUDIT BEFORE SHOWING

Никогда не показывать клиенту dashboard "из коробки" сразу после sync. Сначала:
1. Проверить data quality (counts, NULL fields, аномалии типа "100% при 1 сделке")
2. Идентифицировать что точно vs что под вопросом
3. Подготовить анкету с конкретными вопросами под их данные
4. Калибровка → потом показ
