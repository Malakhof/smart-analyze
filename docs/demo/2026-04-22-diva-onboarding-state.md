## Продуктовые особенности diva (важно для системы)

### Два отдела — два скрипта
- **МОПы** (основной отдел) → скрипт в `2026-04-22-diva-sales-script.md`
- **Первая линия** (Жихарев + Чернышова) → **отдельный скрипт, НЕ прислан** (запросить)

### Кураторы НЕ оцениваются
Список исключения: Лукашенко, Чернышева, Марьяна, Чиркова, Щ, Добренькова, Романова, Довгалева, Николае

⚠️ **Внимание на опечатки:** "Чернышова" (МОП) vs "Чернышева" (куратор) — фамилии почти идентичны, не перепутать при настройке.

### Дашборд РОПа (специфика diva)
Критично показывать:
- Наборы на менеджера
- НДЗ (недозвон)
- Автоответчики
- Реальные разговоры (не путать с автоответчиками)
- Оценка МОП индивидуально + суммарно по отделу

### Критические ошибки (по diva-скрипту)
1. Перебивать клиента
2. Отсутствие выявления потребностей
3. Отсутствие отработки возражений
4. Отсутствие попытки сделки
5. Не назначен следующий шаг
6. Монологная презентация не под боль клиента

### Продажа = любой платный заказ
`Deal.amount > 0 AND status='WON'`. Нет разделения на флагман/трипваер.

### Истинная сделка = клиент × продукт
Один клиент = много карточек если разные email/телефоны. Авто-детект дублей ДА, авто-мёрдж НЕТ (только рекомендация).
