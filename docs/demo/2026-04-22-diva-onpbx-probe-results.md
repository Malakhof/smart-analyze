### 2. История звонков — `POST /mongo_history/search.json`

Параметры: `limit`, `start_stamp_from`, `start_stamp_to` (unix timestamps, 10 цифр)

**За 7 дней = 4 961 звонок.**

Поля:
- `uuid` — идентификатор звонка
- `caller_id_number` — внутренний номер менеджера (100-126)
- `destination_number` — телефон клиента
- `start_stamp` / `end_stamp` — unix-время
- `duration` — общая длительность в секундах
- `user_talk_time` — **время реального разговора** (0 = недозвон, >0 = ответили)
- `hangup_cause` — причина завершения (`NORMAL_CLEARING` / `ORIGINATOR_CANCEL` / `NO_ANSWER` и т.д.)
- `accountcode` — `inbound` / `outbound`
- `gateway` — тип транка

## ❌ ЧТО НЕ ПОЛУЧИЛОСЬ

### Скачивание аудиозаписи по UUID — НЕ НАЙДЕН endpoint

Проверены все стандартные варианты (25+ эндпоинтов). Все возвращают `404 Not Found`:
- `/record.json`, `/record/*.json`
- `/records.json`, `/records/*.json`
- `/mongo_history/record.json` / `get_record.json` / `file.json` / `link.json` / `download.json` / `url.json`
- `/history/record.json`
- `/media/*.json`, `/storage/*.json`, `/archive/*.json`, `/cdr/*.json`
- Прямые URL `/mp3/{uuid}.mp3`, `/records/{uuid}.mp3` на `pbx1720.onpbx.ru` — 307 redirect на marketing-страницу
- Альтернативные хосты: `static.onlinepbx.ru`, `record.onlinepbx.ru` — не отвечают

**Вероятная причина:** либо endpoint имеет нестандартное имя (в документации которую я не смог распарсить полностью), либо audio-ссылки приходят только через webhooks на завершение звонка (`hangup` event), либо включение API-записей требует отдельной активации.

>
> Нужно дословно: формат URL + параметры + как авторизоваться. Пример для uuid `97df4ef7-c251-4935-a835-a9d583c17355` будет идеально.
