## Готовый текст для пересылки

> Таня, привет! Мы (Sales GURU) делаем для diva.school AI-аналитику звонков менеджеров — расшифровки + оценка качества каждого разговора. Чтобы это работало, нам нужен прямой доступ к onlinePBX (телефония), а не только то что приходит через интеграцию в GetCourse.
>
> Помоги пожалуйста с тремя вещами:
>
> **1. API-доступ к onlinePBX**
> Нам нужно, чтобы мы могли автоматически забирать новые звонки и audio-записи. Конкретно:
> - URL API onlinePBX (типа `https://api.onpbx.ru/...`)
> - **API-ключ** (или пара login / password для API)
> - **Account ID** (идентификатор кабинета diva.school в onlinePBX)
> - Если есть webhook для уведомлений о новых звонках — пришли URL и формат
>
> **2. Подтверждение стерео-записи**
> С 10 апреля у вас включена стерео-запись (LEFT=клиент, RIGHT=менеджер) — это критично для нашего AI (точность распознавания ролей 100%). Подтверди пожалуйста что:
> - Стерео реально пишется (можно открыть любой звонок после 10 апреля и проверить — должно быть 2 канала)
> - Стерео включено для **всех** менеджеров (не только новых)
>
> **3. Возможность скачивать audio через API**
> В onlinePBX иногда отдельной галочкой включается «доступ к скачиванию записей через API». Без неё API возвращает ссылку, но скачать нельзя. У одного из наших клиентов из-за этого 14 000 звонков заблокированы — не хочется повторять. Проверь пожалуйста что эта опция активна.
>
> **Зачем это всё:** прямой доступ через API даёт real-time (звонок только закончился — у нас уже расшифровка), а через GetCourse-интеграцию мы получаем только то что они нам пробрасывают, и с задержкой. Плюс через GC мы не видим звонки кураторов отдельно от менеджеров (а нам нужно их разделять). Прямой доступ решает оба.
>
> Если есть документация onlinePBX по API — скинь линк, я разберусь сам. Если нет — можем созвониться на 10 минут с их саппортом.

## Что нам реально нужно (техническое описание для нас, не для Тани)

1. **API endpoint** onlinePBX для:
   - List звонков за период
   - Download audio по id звонка
   - Webhook callback при создании нового звонка (если поддерживается)

2. **Авторизация:**
   - Bearer token / API key / OAuth2
   - Любой формат — разберёмся

3. **Формат audio:**
   - Стерео wav или mp3
   - Левый канал = клиент, правый = менеджер (подтвердить в ответе Тани)

4. **Данные по каждому звонку:**
   - UUID звонка (есть в onlinePBX)
   - Номер клиента
   - Номер менеджера (внутренний + внешний если разные)
   - Дата/время начала и конца
   - Длительность разговора
   - Причина завершения
   - ID сделки в GC (если связано)
