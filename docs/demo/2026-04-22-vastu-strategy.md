## 🎨 ИДЕЯ КЛИЕНТА: "верифицировать звонки из 2 мест"

User: *"в геткурсе меня данные из пбх впечатлили. они прям облегчают жизнь"*

Для diva — двойная верификация: **GetCourse** (контакты, сделки) + **onlinePBX** (полная история звонков с ID и ФИО).

**Для vastu можно ли так же?** Sipuni имеет публичный API (https://sipuni.com/help/api), но требует:
- Sipuni API user (`078268` — есть)
- **Sipuni API secret/key** ← НЕ ЕСТЬ

**Если хотим dual-source для vastu** — запросить у клиента Sipuni API key (одна строка). С ним:
- Прямая история звонков (минуя amoCRM)
- Сверка: всё что Sipuni → попало ли в amoCRM
- Real-time webhook вместо polling

**Cost для клиента:** ноль, обычная функция Sipuni.
**Cost для нас:** ~2 часа на adapter (как onPBX для diva).

→ Включить в финальный список вопросов клиенту: *"Можете дать Sipuni API key? Тогда сделаем такую же двойную сверку звонков как у diva с onPBX — это страхует от потерь и ошибок"*.
