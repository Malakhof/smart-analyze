# Killer Features — фичи которые отделят нас от конкурентов

## Feature 1: Автогенерация скрипта продаж по сайту клиента

**Суть:** Клиент кидает URL своего сайта → AI парсит сайт, понимает продукт/услугу → автоматически генерирует скрипт продаж → этот скрипт сразу идёт в модуль контроля качества для проверки звонков.

**Поток:**
```
URL сайта → парсинг (описание, УТП, цены, FAQ) 
→ DeepSeek генерирует скрипт (чеклист шагов)
→ Script автоматически создаётся в системе
→ Все звонки проверяются по этому скрипту
```

**Ценность:** Клиент не придумывает скрипт сам. Через 5 минут после подключения у него уже работает контроль качества.

**Реализация:**
- Парсер сайта (cheerio/puppeteer → текст)
- DeepSeek промпт: "На основе сайта создай скрипт продаж из 8-12 пунктов"
- API: POST /api/scripts/generate-from-url
- UI: кнопка "Сгенерировать скрипт из сайта" в настройках скриптов

## Feature 2: Ежедневная выжимка менеджеру

**Суть:** В конце дня (или недели) менеджер получает персональный отчёт: что сделал хорошо, что можно улучшить, конкретные рекомендации по его звонкам за день.

**Поток:**
```
Конец дня → собрать все звонки менеджера за день
→ DeepSeek анализирует пакет
→ Генерирует: ✅ Плюсы | ❌ Минусы | 💡 Рекомендации
→ Отправляет в Telegram менеджеру (или на email)
```

**Ценность:** Не только РОП видит проблемы — менеджер сам получает обратную связь и растёт. Самообучающийся отдел продаж.

**Реализация:**
- Cron job (ежедневно 18:00): собрать звонки за день по каждому менеджеру
- DeepSeek промпт: "Проанализируй 5 звонков менеджера, дай плюсы/минусы/рекомендации"
- Telegram bot: отправка персонального отчёта каждому менеджеру
- UI: настройка расписания и получателей

## Feature 3: Промотка аудио по словам из расшифровки

**Суть:** Кликаешь на любое слово в расшифровке → аудио перематывается на это место и воспроизводится.

**Поток:**
```
Whisper API с timestamps=true → word-level timestamps
→ Каждое слово кликабельно
→ Клик → audio.currentTime = word.timestamp
→ Воспроизведение с этого места
```

**Ценность:** РОП может мгновенно найти проблемное место в разговоре, не слушая весь звонок.

**Реализация:**
- Whisper API: добавить `timestamp_granularities: ["word"]` в запрос
- Хранить: `{word: "Здравствуйте", start: 0.0, end: 0.45}` в Message.metadata (JSON)
- UI: каждое слово в `<span onClick={() => audio.currentTime = start}>`
- Подсветка текущего слова при воспроизведении (как караоке)

## Feature 4: AI-агент — чат-помощник РОПа (KILLER FEATURE)

**Суть:** На сайте встроенный чат с AI-агентом. Пишешь вопрос → агент анализирует ВСЕ данные (звонки, сделки, воронку, паттерны) → отвечает инсайтами.

**Примеры запросов:**
- "Почему конверсия упала на этой неделе?"
- "Кто из менеджеров сливает больше всего?"
- "Что не так с Екатериной?"
- "Покажи лучшие практики закрытия в этом месяце"
- "Сколько денег мы потеряли из-за медленных ответов?"

**Поток:**
```
Вопрос пользователя
→ RAG: запрос к БД (сделки, анализы, паттерны, звонки)
→ Контекст + вопрос → DeepSeek
→ Ответ с конкретными цифрами, именами, рекомендациями
```

**Ценность:** 
- НЕ НУЖНО лазить по сайту и соединять данные в голове
- Верхний слой аналитики — инсайты поверх всех данных
- CEO/директор может заменить РОПа для маленьких команд (2-3 менеджера)
- **Главное отличие от конкурентов** — у них статичные дашборды, у нас живой агент

**Реализация:**
- UI: чат-виджет в правом нижнем углу (или отдельная вкладка "AI Помощник")
- Backend: API route POST /api/chat
- RAG pipeline: вопрос → определить какие данные нужны → SQL запросы → контекст → DeepSeek → ответ
- Функции агента (tool use): getDealStats(), getManagerInfo(), getPatterns(), getCallAnalysis()
- История чата в БД (ChatMessage model)

**Позиционирование:** "Не дашборд — а AI-помощник РОПа. Спросил — получил ответ."

## Feature 5: Лидген из HH.ru (машина уже готова!)

**Суть:** Парсим вакансии "менеджер по продажам", "РОП", "директор по продажам" на HH.ru → это компании с отделами продаж → звоним с предложением.

**Скрипт звонка:**
"Здравствуйте! Видим, что вы набираете менеджеров — значит отдел продаж растёт. Хотите бесплатно проанализировать 50 сделок и увидеть, где теряете деньги?"

**Флаги-триггеры для звонка:**
- Набирает менеджеров = растёт/проблемы с текущими
- Меняет РОПа = проблемы в управлении
- "Падение продаж" в описании = горячий лид
- Много вакансий менеджеров = текучка = проблема

**Реализация:** Уже есть в QUP docs (Lead Monitor Bot, HH.ru парсер). Переиспользуем.
