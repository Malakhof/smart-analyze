**Date:** 2026-04-23
**Source:** прямой API onPBX `mongo_history/search.json` + `user/get.json`
**Sample:** 8 447 звонков diva за 13 дней (10-22.04)

## Доступные insights (из метаданных без транскрипции)

### Раздел 1: Воронка дозвонов (Pickup funnel)

```
8 447 попыток → 5 005 поднятых (59%) → 1 780 real talks (21%)
```

- **3 442 (41%)** — никто не поднял
- **3 225 (38%)** — короткий разговор <30s (автоответчик/ошибочный/сразу повесили)
- **1 780 (21%)** — реальные продажи

**Hangup причины (6 908 NORMAL_CLEARING + остальные):**
- 1 909 NORMAL_CLEARING при talk=0 = клиент не взял трубку
- 635 NORMAL_TEMPORARY_FAILURE = линия упала
- 465 ORIGINATOR_CANCEL = менеджер сбросил сам (early cancel)
- 108 USER_BUSY = занято
- 106 UNALLOCATED_NUMBER = несуществующий номер (плохая база!)

## Что доступно через API onPBX

| Endpoint | Status | Использование |
|---|---|---|
| `/auth.json` | ✅ | Получение session key |
| `/mongo_history/search.json` | ✅ | Все звонки + download=1 для аудио |
| `/user/get.json` | ✅ | 27 МОПов: ext + ФИО + delay-настройки |
| `/group/get.json` | ⚠️ | Возвращает status=0 (пусто или нет тарифа) |
| `/queue/`, `/ivr/`, `/schedule/`, `/did/`, `/blacklist/`, `/voicemail/`, `/callback/` | ❌ 404 | Не на этом тарифе onPBX |

**Резюме:** на текущем тарифе доступны history + users + audio. Этого хватает на ВСЁ что описано выше.

## Стоимость / время

- **Backend для всех insights:** ~3-4 часа кода (агрегации SQL + onPBX API caller)
- **Frontend (страница /diva-insights):** ~4-5 часов (графики через recharts уже стоит)
- **Cost runtime:** **0₽** — данные уже у нас, без LLM
- **Update frequency:** ежедневный sync onPBX (cron ~5 мин/день)

**Без транскрибации можно поставить готовые insights в платформу за 1-2 дня.**

После транскрибации добавляются Sales Phrases / Script Score / Tone — это ещё ~3 дня UI.
