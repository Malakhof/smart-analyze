# Стратегия масштабирования Smart Analyze / SalesGuru

## Текущее состояние (MVP)
- 1 сервер Timeweb (4GB RAM, без GPU)
- Whisper через API (OpenAI) или с мака
- DeepSeek V3 API
- Хватает для демо и первых 1-5 клиентов

## Модель масштабирования: Группы по 50 клиентов

Каждая группа = отдельный сервер с полным стеком (App + DB + WhisperX).
Полная изоляция, линейное масштабирование, простой деплой.

```
Группа 1 (50 клиентов)          Группа 2 (50 клиентов)
┌─────────────────────┐          ┌─────────────────────┐
│  App + DB + Whisper  │          │  App + DB + Whisper  │
│  1 GPU сервер        │          │  1 GPU сервер        │
│  sa1.salesguru.ru    │          │  sa2.salesguru.ru    │
└─────────────────────┘          └─────────────────────┘
```

### Преимущества модели
- Нет Kubernetes, балансеров, shared DB
- Падение 1 сервера = проблемы только у 50 клиентов
- Деплой: `git pull && docker compose up`
- Масштабирование: +1 сервер = +50 клиентов

## Требования к серверу (1 группа = 50 клиентов)

### Для WhisperX self-hosted

| Параметр | Минимум | Оптимум |
|----------|---------|---------|
| GPU | RTX 3060 12GB | RTX 3090 24GB / A4000 16GB |
| RAM | 16 GB | 32 GB |
| CPU | 4 cores | 8 cores |
| SSD | 100 GB | 200 GB |
| Цена | ~8-12K₽/мес | ~15-25K₽/мес |

### Производительность WhisperX

WhisperX large-v3 на RTX 3090 = ~30x realtime:
- 1 мин звонка = 2 сек обработки
- 100 звонков × 5 мин = 17 минут обработки
- 500 звонков × 5 мин = 1.5 часа

## Когда переходить на Kubernetes

Только при 1000+ клиентов, когда:
- 20+ серверов сложно менеджить вручную
- Нужен автоскейлинг GPU нод
- Нужен zero-downtime deployment

До этого — группы по 50 на отдельных серверах.

## SSL / Домены

- Cloudflare Origin Certificate (15 лет, *.qupai.ru)
- Каждая группа: sa-{N}.salesguru.ru или кастомный домен клиента
- Cloudflare proxy для DDoS защиты и CDN
