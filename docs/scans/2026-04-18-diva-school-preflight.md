## Ключевые находки

### 1. Масштаб: enterprise-класс
diva.school — это **в 1500 раз больше Reklama по заказам** (966K vs 295/90д). Школа реальный enterprise.

### 2. Готовая структура РОП
В фильтрах звонков уже есть готовые сегменты:
- "Звонки оп1" (отдел продаж 1)
- "ОП. Звонки свыше 5 минут"
- "РОП. Звонки ОП3."
- "Звонок состоялся"

→ калибровка нашей системы под их структуру будет естественной.

### 3. Маркетологи ставят теги/UTM грамотно
Сегменты заказов содержат: TG_ADS, PIN_ADS, UTM-кампании, "Был на вебе", "Заказы без UTM-меток".
→ они уже умеют считать данные и ценят аналитику.

### 4. Звонки фильтруются по типу
Поле `direction = income/outcome` доступно. Из 982K записей "контактов" реальные **звонки** = подмножество (которое мы ещё посчитаем).

### 5. Воронки CRM = SPA (Vue.js)
Структура воронок (этапы, названия) грузится через AJAX. HTML-парсинг не вернёт. Для извлечения потребуется:
- либо **CSV экспорт** (стандартный для GetCourse: на любом list-page кнопка "Экспортировать")
- либо **API-эндпоинт** (надо найти в JS bundle)

### 6. Страница statistics — нужен бэкграунд
`/pl/tasks/task/stat` отдаёт таймаут 30s. Аналог — `/pl/sales/dealstat/index` (это работает, 287KB) → там есть готовые отчёты по продажам с группировкой по менеджерам.
