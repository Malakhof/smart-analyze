## 1. Что МОЖНО делать без транскрипции (после sync)

| Категория | Что показываем клиенту | Источник в БД |
|---|---|---|
| **Рейтинг менеджеров** | Выручка, WON/LOST, конверсия, средний чек, цикл сделки | Deal + Manager |
| **Карта воронки** | Сколько сделок на каждом этапе, визуальный bottleneck | Deal.currentStageCrmId + FunnelStage |
| **Где застревают сделки** | Среднее время на каждом этапе по воронке | DealStageHistory |
| **Нагрузка менеджеров** | Открытые / просроченные / выполненные задачи per manager | Task |
| **Время ответа клиенту** | Дельта между сообщениями клиент→менеджер | Message.timestamp |
| **Паттерны переписки** | DeepSeek на Message.content — что пишут топы, что середняки | Text messages |
| **Частота коммуникации** | Сколько писем/звонков per deal, per менеджер | Message + CallRecord counts |
| **Длительность звонков** | Avg duration per manager, peaks | CallRecord.duration (только метаданные) |
| **Пиковые часы** | Когда максимум активности команды | timestamps всего |
| **Причины отказов** | На каком этапе чаще всего LOST, loss_reason | Deal.status + DealStageHistory |
| **Аудио-архив** | Показать ссылку `audioUrl` в UI — клиент клик → слушает сам | CallRecord.audioUrl |

**Вывод:** Весь AI-дашборд, все метрики РОПа, все паттерны по переписке — **без единого звука Whisper**.

## 4. GetCourse READ-ONLY Safety Protocol

### Контекст опасности

- У нас админ-доступ (`malakhoff@gmail.com`, isAdmin=true, isManager=true)
- Один неверный клик = потерянные данные клиента, рассылка писем, удаление пользователей
- Проёб = потеря доверия клиента и потенциальной сделки
- **Zero tolerance к ошибкам**

### Предохранитель #0: Dry-Run Protocol

**ПЕРЕД любым действием на GetCourse:**
1. Я показываю **список URL которые собираюсь дёрнуть**
2. Ты подтверждаешь конкретный набор
3. Я дёргаю **по одному** → показываю что получил
4. Переходим к следующему только после твоего "ок"

**Никакого автопилота.** Полный ручной контроль на каждом шаге.

### Предохранитель #1: Только HTTP GET

**РАЗРЕШЕНО:**
```bash
curl -X GET  # только GET
```

**СТРОГО ЗАПРЕЩЕНО:**
```bash
curl -X POST   # ❌
curl -X PUT    # ❌
curl -X DELETE # ❌
curl -X PATCH  # ❌
```

Никакие "безобидные" POST-запросы. Даже если URL выглядит readonly — форма может вызвать side effect.

### Предохранитель #2: URL Whitelist

Только эти пути (ТОЛЬКО GET):

```
/pl/user/contact/index           # список звонков
/pl/user/user/index              # список пользователей
/pl/cms/deal/index               # сделки
/pl/cms/deal/stat                # статистика сделок
/pl/user/user/statistic          # метрики
/pl/user/message/index           # сообщения (если доступен)
/pl/cms/pipeline/index           # воронки продаж
```

**Ничего вне этого списка** без явного подтверждения.

### Предохранитель #3: Blacklist URL-паттернов

**НИКОГДА не ходим на:**

| Паттерн | Почему |
|---|---|
| `?action=*` | может триггернуть действие |
| `?delete=*` | удаление |
| `?remove=*` | удаление |
| `?send=*` | отправка (письма, SMS) |
| `?approve=*` | подтверждение |
| `?create=*` | создание |
| `/admin/*` | админ-действия |
| `/setting/*` | изменение настроек |
| `/integration/*` | интеграции (могут разломать интеграции клиента) |
| `/webhook/*` | вебхуки |
| `/notification/*` | уведомления |
| `/billing/*` | биллинг |

### Предохранитель #4: Результаты в /tmp, не в нашу БД

Pre-flight не пишет в нашу БД. Всё скачанное:
- Сохраняется в `/tmp/gc-scan-<timestamp>.json`
- Агрегируется в отчёт-анкету
- БД остаётся нетронутой

Если что-то неожиданное — откатываем через `rm /tmp/gc-*`.

### Предохранитель #5: Никакого Playwright в pre-flight

Playwright может случайно кликнуть "Удалить" если появится диалог. Или submit форму. Или выполнить JavaScript.

**Для pre-flight — только curl/fetch**. Никакого browser automation.

### Предохранитель #6: Rate-limit сам себе

- Max 1 запрос/секунду
- Max 20 запросов за сессию
- Если GetCourse вернул что-то странное (503, rate limit, unexpected JSON) → СТОП, ждём ручного разбора

### Предохранитель #7: Что делаем если "агент предложил" что-то не-whitelist

```
Response: СТОП. URL не в whitelist. Прошу подтверждения от человека.
Ответ пользователя: "да, можно" → добавляем в whitelist
Ответ пользователя: "нет" / молчание → не делаем
```

### Предохранитель #8: Логирование

Каждый запрос логируется в `/tmp/gc-scan-log.txt`:
```
[2026-04-18 19:00:00] GET /pl/user/contact/index → HTTP 200, 12KB
[2026-04-18 19:00:02] GET /pl/cms/deal/stat → HTTP 200, 3KB
```

Если после скана что-то у клиента "странное" — можем показать ровно что мы делали.

### Предохранитель #9: Cookie не ротируем

Наш cookie у diva.school — не обновляем, не пересоздаём, не релогинимся. Используем ровно тот что уже есть в `CrmConfig.gcCookie`. Любой re-login = риск триггернуть admin notification.

## 5. Права клиента — что просить для продакшн онбординга

На основании нашего опыта с diva.school:

### Уровни прав GetCourse

| Роль | Что видит | Достаточно нам? |
|---|---|---|
| **Менеджер** | Свои клиенты + свои сделки | ❌ не видит общую стат |
| **Руководитель отдела / CRM** | Все сделки в CRM + отчёты + пользователи | ✅ **Этого хватит** |
| **Администратор** | Всё + настройки + интеграции | избыточно, опасно |

### Рекомендация для клиентов в анкете

Создать **отдельного юзера специально для SalesGuru**:
- Имя: `SalesGuru-bot`
- Email: `salesguru@<domain>.ru` (можно alias)
- Роль: **Руководитель отдела продаж** (видит CRM + пользователей + отчёты)
- **НЕ** Администратор (если клиент предложит — вежливо отказываемся)

Для первых клиентов (где уже админ) — добавляем в анкету "на будущее переведём на отдельного юзера".

### Запросы по whitelist (с подтверждением каждого)

5. `GET /pl/user/contact/index?limit=1&created_at[from]=<90d ago>` — **кол-во звонков за 90 дней**
6. `GET /pl/user/user/index?limit=1&role=manager` — **кол-во менеджеров**

## 7. Стратегические принципы (ставим в память)

1. **Клиентские данные священны.** Мы — гости в их системе. Лучше упустить фичу, чем сломать им CRM.
2. **Read-only по умолчанию.** Любой запрос требует подтверждения "это GET? это безопасно?".
3. **Транскрипция ≠ Фаза 2.** 80% ценности — без Whisper.
4. **Pre-flight обязателен** для каждого нового клиента. Без анкеты никогда не начинаем sync.
5. **Whitelist URL** для каждой платформы. CRUD через blacklist-проверки.
6. **Log everything.** Если клиент спросит "что вы делали" — можем показать ровно.
